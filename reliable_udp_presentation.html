Project Overview

Problem Statement

UDP is a connectionless protocol that provides no guarantees for delivery, ordering, or duplicate elimination. This project implements a reliable file transfer system on top of UDP by building reliability mechanisms at the application layer.

Educational Purpose

Developed as part of a Computer Communication Networks course to:

  • Understand transport layer protocols (UDP vs TCP)
  • Implement reliability mechanisms manually
  • Gain hands-on socket programming experience
  • Apply theoretical networking concepts to practice

Reliability Mechanisms Implemented

Guaranteed Delivery Correct Ordering Duplicate Elimination Automatic Retransmission Timeout Handling

Computer Networking Concepts

1. UDP Socket Programming

  • Connectionless communication using sendto() and recvfrom()
  • Port binding and address management
  • Client-server architecture

2. Reliable Data Transfer

Stop-and-Wait Protocol:

  • Send one frame, wait for acknowledgment
  • Only proceed after receiving correct ACK
  • Retransmit on timeout or wrong ACK

Technical Implementation Details

Reliability Mechanisms

1. Acknowledgment (ACK)

Receiver sends frame ID back to confirm receipt. Sender verifies ACK matches sent frame.

2. Timeout Handling

2-second timeout using setsockopt(SO_RCVTIMEO). Triggers retransmission if no ACK received.

3. Sequence Numbers

Each frame has unique ID starting from 1. Prevents duplicates and ensures ordering.

Performance Characteristics

Metric Value
Frame Size 2048 bytes data + 16 bytes header
Timeout 2 seconds
Max Retries (Handshake) 20 attempts
Max Retries (Data) 200 attempts per frame
Throughput (Localhost) ~2 MB/s (theoretical)
